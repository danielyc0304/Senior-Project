# 機器學習量化交易專題 - 代碼指令

## 應用範圍

這些指令適用於整個專題的所有代碼，包括 Python 腳本（main.py 等）和 Jupyter Notebook（lstm.ipynb）。

## 代碼風格與規範

### Python 代碼風格

- **遵循 PEP 8 標準**：使用 4 個空格縮進，變量/函數名使用 snake_case，類名使用 PascalCase
- **類型提示 (Type Hints)** 是必須的：
  - 所有函數必須包含參數類型和返回值類型
  - 例：`def fetch_stock_data(ticker: str, interval: str = "15m") -> pd.DataFrame:`
  - 使用 `from typing import` 導入必要的類型（List, Dict, Tuple, Optional 等）
- **文檔字符串 (Docstrings)** 格式：

  ```python
  def calculate_sma(prices: np.ndarray, window: int) -> np.ndarray:
      """
      計算簡單移動平均 (SMA)。

      Args:
          prices: 價格數組
          window: 計算窗口大小

      Returns:
          SMA 值的 numpy 數組
      """
  ```

## 數據處理最佳實踐

### 1. 數據驗證與清理

- **在處理前驗證**：檢查缺失值、異常值、數據類型
- **記錄變化**：數據清理前後的記錄數，添加日誌輸出
- 例子：檢查 NaN、重複行、邊界值

### 2. 訓練/測試數據分離

- **始終分離訓練和測試數據**，避免數據泄露（data leakage）
- 使用時間序列適當的分割方式（不能隨機洗牌）
- 建議比例：訓練 70%、驗證 15%、測試 15%（時間序列順序）
- 在函數簽名中明確說明返回值對應的數據集

### 3. 技術指標計算

- 使用 numpy/pandas 實現技術指標（SMA, EMA, WMA 等）
- 每個指標應是獨立的函數，易於測試和重用
- 包含邊界條件處理（例如：初始 NaN 值的處理）

## 輸出與結果管理

### 1. 文件路徑約定

- **所有輸出必須保存到 `output/` 資料夾**
- 文件名必須包含**時間戳記**，格式：`YYYYMMDD_HHMMSS_描述.ext`
- 例：`20260317_143022_backtest_results_2330.csv`

### 2. 輸出文件類型

- **CSV 文件**：回測結果、交易信號、性能指標（編碼：UTF-8）
- **圖表**（matplotlib/seaborn）：
  - 價格與移動平均線對比
  - 買賣信號的標記
  - 累積收益曲線
  - 模型性能指標（仅適用於 ML 模型）
  - 保存格式：PDF（高質量）或 PNG（快速查看）
- **模型文件**：pickle or joblib（適用於 scikit-learn 模型）

### 3. 結果記錄

- 使用日誌或輸出視頻記錄關鍵步驟（數據準備、訓練進度、回測結果）
- 每個實驗應記錄參數、開始/結束時間、最終性能指標

## 測試與驗證

### 1. 數據驗證邏輯

- 檢查股票數據完整性（無缺失的交易時段）
- 驗證技術指標計算（與已知值對比或單元測試）
- 驗證數據時間順序（確保無重複或亂序）

### 2. 錯誤處理

- 使用 try-except 處理異常：
  - 網絡異常（Yahoo Finance 連接失敗）
  - 數據異常（無效的股票代碼、數據格式錯誤）
  - 文件異常（輸出目錄不存在、寫入權限問題）
- 錯誤消息應明確指出發生地點和原因

### 3. 單元測試（可選但推薦）

- 為核心函數（指標計算、信號生成）編寫簡單測試
- 測試文件名：`test_*.py`（如 `test_indicators.py`）

## 量化交易特定指南

### 1. 股票數據收集

- 使用 `yfinance` 從 Yahoo Finance 獲取數據
- 記錄數據來源、時間範圍、間隔（15m、1h、1d 等）
- 保存原始數據副本以便重現結果

### 2. 買賣信號

- 信號應清晰定義（例如：1 = 買入、-1 = 賣出、0 = 持有）
- 記錄信號生成的邏輯和時間點
- 避免未來信息泄露（不能使用未來數據生成信號）

### 3. 回測評估指標

- **必須包含**：總收益率、年化收益率、夏普比率、最大回撤
- **推薦包含**：勝率、平均盈虧比、交易次數、手續費影響
- 與買賣持有策略（Buy & Hold）進行基準對比

## Notebook 特定指南

### 1. 單元組織

- 將不同功能分離到不同的單元（數據準備、特徵工程、模型訓練、評估）
- 每個單元應有清晰的 Markdown 註釋說明其目的
- 最後一個單元保存結果到 output/

### 2. 可重現性

- 在第一個單元設置隨機種子（numpy, sklearn, pandas）
- 在每個單元開頭導入必要的包，避免跳過單元執行

### 3. 輸出顯示

- 使用 `print()` 和 `display()` 呈現關鍵結果
- 生成並顯示圖表（之後保存到 output/）
